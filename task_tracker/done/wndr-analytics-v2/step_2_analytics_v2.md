## Задача

Обновить `analyze_community.py` в трёх направлениях:

### А) Эвристика "содержательный пост vs обсуждение"

Порог по символам: пост считается "содержательным" если `char_count >= 150`.
Логика: короткие сообщения ("ого!", "👍", "когда старт?") — обсуждение.
Длинные — реальный контент (оффер, коммит, запрос, интро).

Добавить в каждое сообщение при анализе флаг `is_substantial = char_count >= 150`.

Новые счётчики на пользователя:
- `substantial_posts` — содержательных постов всего
- `substantial_by_topic` — по каждому топику отдельно

### Б) Топы по каждому контент-топику

Для каждого из топиков: offerings, requests, sales, commits, harvest, daily, intro, together —
вывести топ-10 авторов по числу **содержательных постов** в этом топике.

Формат в отчёте:
```
## Топ по топику: Запросы
| # | Участник | Содержательных постов | Всего сообщений |
```

### В) Динамика активности по датам

Три таблицы:
1. Сообщений по месяцам (уже есть — оставить)
2. Топ-10 участников + их активность по месяцам (матрица: строка=участник, колонка=месяц)
3. Активность по неделям (уже есть — оставить)

### Г) Реакции — топ постов

Из всех топиков выбрать топ-20 постов по суммарному числу реакций.
Показать: автор, топик, начало текста, суммарно реакций, список эмодзи с количеством.

## Хранение данных

Оставляем JSON-файлы как есть — это достаточно для аналитики.
Если в будущем нужна веб-визуализация — генерировать отдельный `analytics_data.json`
(плоская структура специально для Chart.js).

Добавить флаг `--data-json data/exports/wndr/analytics_data.json` в analyze_community.py —
сохраняет структуру для фронта (опционально, не блокирует основной отчёт).

## Критерии готовности
- [ ] В отчёте есть топ-10 по каждому контент-топику
- [ ] Посты разделены на "содержательные" (>=150 символов) и короткие
- [ ] Топ-20 постов по реакциям присутствует (или сообщение "реакций нет" если все 0)
